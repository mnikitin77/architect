Задание 2. Выбрать архитектуру домашнего CRM или ERP приложения. Обосновать выбранное решение.
==================================================================================================================
Для домашнего CRM приложения я выбираю набор из разных архитектур.

1) Стандартные бизнес-модули CRM системы (База клиентов, Продажи, Обслуживание, Маркетинг) будут распределены
по отдельным микросервисным приложениям и будут взаимодействовать между собой напрямую по API (например, REST).
Сердце и позвоночник CRM системы, клиентская база, назовём CRM Core будет хранить и управлять сущностями, 
описывающими клиентов юрлиц и физлиц (Организация, Контакт, Адрес и т.д.), а также базу взаимодействий с клиентами
(Activity) по всевозможным каналам (телефон, email, чат и т.п.). Остальные CRM-модули будут реализовываться по
мере потребности, предварительно как микросервисы.

2) Внутри каждый модуль системы (микросервис) будет реализован очевидно в многослойной архитектуре с целью сделать
систему более гибкой, поддерживаемой и надёжной. Слои будут служить для изоляции себя друг от друга для реализации
вышеописанных целей. Основные слои:
- UI (уровень представления)
- Бизнес-логика
	- DTO
	- Service
	- DAO (Repository, ORM Entities)
- РСУБД

3) Если CRM выростет из домашней в корпоративную, и это будет иметь смысл, возможно, для интеграции модулей между 
собой и с другими системами (бухгалтерия, склад, BI и т.п. AI =) ) будет внедрена ESB и взаимодействия между 
модулями пойдут через неё.
